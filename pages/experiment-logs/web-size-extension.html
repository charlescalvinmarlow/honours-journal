Something Material Honours Journal

Web Size Extension

return

Aims
  • To try and see how much data is being transferred across the network as you use
  • To test how a browser extension could be a design solution
  • To see how LLMs might be used to quickly test simple prototypes
Precedents / Context
  • Low Tech Magazine. (How this website has to lean of file sizes to function)
  • Demoscene. (How these projects transfer very little code over the network, but can be quite demanding on device energy usage)
  • Permacomputing
  • Digital Sobriety
Process / Methods

Basically it was a little thought I had while I needed a break from writing. What if every kb of data that came down on to your computer across the internet was tracked and shown to you? As I have decided to experiment with LLMs in a light and methodical way, I figured this was a good chance to see what it could do.

I chose Gemini, because it's free, Google has lots of data and honestly its too big to fail. I simply asked "can you write a firefox plugin that displays the size in kb of a website". I did not know what language was needed, how to compile and make this thing work, if it was something I could test, I simply let the LLM handle all those contextual details. In this one prompt I got a working prototype. I tried prompting a couple other small changes, but they ended up not working, and decided this was enough for this small project.

Reflection on Action

I was impressed at how well the LLM interpreted what I was asking for. Not only did it make a simple small prototype, but it provided explanations for how it worked, how to test it in Firefox, and also limitations and technical details that might mean this prototype might not be 100% accurate for what I am trying to do. Even with these 2 or 3 prompts, I was even directed to some information I was not aware of, terms like Digital Sobriety, organisations like The Shift Project which were defining ways to calculate data to energy usage.

I think overall, despite the glaring hypocrisy of using an energy hungry technology to prototype an energy conscious one, this felt like a pretty good experiment with LLMs. It was small and low scope enough that a couple of prompts got what I needed out of it, it let me explore technologies and coding language I don't know and it gathered additional context around those technologies to help me learn about them. Understanding that I just want a *prototype* from the LLM was I think important. It's not pretty, I don't need Gemini to tweak any of the details, I just want to see how the technical side can work, it did this quickly and easily and there was no need to continue using the LLM after that.

Reflection for Action

I think there is something to this idea of using browser extensions. As Zoe said, demonstration is powerful, make these invisible details of your internet experience visible and in your face and you can better understand them. Perhaps it is an uncomfortable truth and it would be hard to convince actual end users to install something like this, so maybe it should be designed for a specific space, for some kind of exhibition or demonstration?

Possibilities for future action:

  • Research energy or data conscious browser extensions
  • Design an interface for an extension (just visual not functioning)
  • More research on how the Internet works
